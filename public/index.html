Build your own agent

Two kinds of agent, both small and grounded by construction. No framework to learn — just a prompt or a recipe.

1 · A read agent = a prompt

Retrieve grounded chunks, then ask the model to do something with them. Add a function in src/agents/prompts.ts and a method on the Brain that reuses retrieve → generate — it inherits citations and the refuse-or-cite contract for free.

// src/agents/prompts.ts
export function buildRiskPrompt(company, context) {
  return `Flag relationship risks for ${company}.
Use ONLY the context. Cite each fact. If nothing, say so.

CONTEXT:
${context}`;
}

// src/brain/brain.ts — reuse the grounded draft() helper
async riskScan(company, scopes) {
  return this.draft(company, buildRiskPrompt(company, ''), scopes);
}

2 · A write agent = an action recipe

Anything with a side effect is a recipe: a draft step + an executor. It’s automatically grounded, human-approved, idempotent, and audited. Copy examples/custom-action.example.ts.

// a recipe = propose (draft, grounded) + execute (the side effect)
const recipe = {
  kind: 'slack_nudge',
  propose: (brain, company, scopes) =>
    brain.draft(company, 'Draft a Slack nudge…', scopes),
  execute: async (action) => slack.post(action.payload),
};
// → proposeAction() → human approves in this console → executor runs once
The console you’re using drives these over HTTP. Your front end, a Slack bot, or a cron job can call the same endpoints — the governance (grounding · approval · idempotency · audit) lives in the core, not the caller.
