Prefer code? Embed it

the same seams, as a library

import { Brain } from 'open-company-brain' — call brain.ingest() / brain.ask() / brain.draft() directly, or createApp() to mount the whole API. A read agent is an instruction + a retrieval query through draft(); a reaction agent is the same, run automatically on ingest.

Everything here runs over the same HTTP API — a front end, a Slack bot, an n8n workflow, or a cron job calls it identically. The governance lives in the core, not the caller.
